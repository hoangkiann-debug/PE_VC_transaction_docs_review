# Matter Profile and Confidentiality

Use a matter profile when the user is reviewing a real transaction package.

## Matter Profile Fields

```markdown
# Matter Profile

- Matter:
- Matter ID / workspace:
- User role / review purpose:
- Client side:
- Counterparty / target:
- Structure:
- Governing law:
- Dispute forum / arbitration seat:
- Round:
- Review scope:
- Output mode:
- Primary / controlling language:
- Client risk preference:
- Key redlines:
- Acceptable fallbacks:
- Confidentiality level:
- Cross-matter use allowed: no / yes, scope:
- Connector status: legal / entity / document:
- Legal-authority cutoff / last checked:
- Assumptions authorized by user:
- Version-chain baseline:
- Documents:
- Missing documents:
- Unreadable / OCR-required documents:
- Review gate status:
```

## Confidentiality Rules

- Default: do not read other matters or use other project-specific redlines.
- If comparing prior deals, ask or confirm that cross-matter comparison is allowed.
- Do not reveal names, economics, or terms from other deals unless the user expressly asks and the materials are in scope.
- When using old deal files only for testing the skill, summarize behavior and do not disclose sensitive deal terms in final chat unless needed.

## Matter Closure

If maintaining a matter workspace in the future:

- Archive rather than delete.
- Keep issue logs and assumptions with the matter.
- Record whether outputs are report, comment plan, redline, or both.
